## この章のゴール

- 自分専用のブランチを作成できる
- なぜ `main` で直接作業しないかを説明できる
- 現在いるブランチを確認できる

## ブランチって何?

ブランチは「**作業場所のコピー**」だと思ってください。

たとえば `main` ブランチで暮らしているリポジトリで、あなたが「自己紹介を書く」という作業を始めたいとします。このとき、いきなり `main` でファイルを編集するのではなく、**`main` の状態をコピーした別の作業場所** を作って、そこで作業するのが Git のお作法です。

```
main ─────────────────────────────────●─── (他の人の変更がここに入る)
       │
       └── add-self-introduction-octocat ─●─●─ (あなたの作業はこっち)
```

作業が終わって「これでよし」となったら、自分のブランチの内容を `main` に合流させます(これがマージです)。これは次の章以降でやります。

## なぜ main で直接作業しないのか

理由は 3 つあります。

1. **他の人と同時に作業できる** — 全員が `main` で作業すると、誰かの変更で他の人の作業が壊れます。ブランチを分ければ、各自の作業は完全に独立しています
2. **失敗してもロールバックしやすい** — ブランチを丸ごと捨てれば、`main` には何も影響しません
3. **レビューしてからマージできる** — ブランチの内容を Pull Request として提出し、第三者の OK が出てから初めて `main` に入ります

> :::message
> 実務でも同じです。多くのチームでは `main` ブランチに直接 commit することは禁止されており、必ずブランチを切って PR 経由でマージするルールになっています。
> :::

## 現在地を確認する

まず、自分が今どのブランチにいるかを確認しましょう。

```bash
git branch
```

実行結果はこんな感じになるはずです。

```
* main
```

`*` がついているのが現在いるブランチです。clone した直後なので `main` にいます。

## ブランチを作って移動する

それでは、自分用のブランチを作って移動しましょう。コマンドは `git switch -c <ブランチ名>` です。

```bash
git switch -c add-self-introduction-<あなたの github-id>
```

例えば GitHub ID が `octocat` なら:

```bash
git switch -c add-self-introduction-octocat
```

成功するとこう出ます。

```
Switched to a new branch 'add-self-introduction-octocat'
```

### コマンドの意味

| 部分 | 意味 |
|---|---|
| `git switch` | ブランチを切り替えるコマンド |
| `-c` | 「create」の c。指定したブランチが無ければ新しく作る |
| `add-self-introduction-octocat` | ブランチの名前 |

> :::message
> 古い資料では `git checkout -b <ブランチ名>` というコマンドを見かけることがあります。これでも同じことができますが、`checkout` は他の用途(ファイルを元に戻すなど)も兼ねていてややこしいので、新しい Git では **ブランチ操作専用の `switch`** が推奨されています。本ハンズオンでも `switch` で進めます。
> :::

## ブランチ名のルール(今日のハンズオン版)

ブランチ名には決まった命名規則があるわけではありませんが、チームで統一されていることが多いです。今日は次のルールでお願いします。

```
add-self-introduction-<あなたの github-id>
```

- `add-self-introduction-` の部分は固定
- `<あなたの github-id>` は実際のあなたの ID に置き換える(例: `octocat`)
- 全部小文字、単語の区切りはハイフン (`-`)、空白や日本語は使わない

> :::message
> 実務でよくあるブランチ名の例: `feat/login-form`, `fix/typo-in-readme`, `chore/update-deps` など。「種類/具体的な内容」のような形式です。今日はシンプルに「動詞-内容-名前」の形にしています。
> :::

## 確認しよう

もう一度 `git branch` を実行してください。

```bash
git branch
```

こうなっているはずです。

```
* add-self-introduction-octocat
  main
```

`*` が新しいブランチに移っていれば成功です。

## ありがちなつまずき

### Q. 間違ったブランチ名で作ってしまった

ブランチ名を変更できます。

```bash
git branch -m add-self-introduction-correct-name
```

`-m` は move(rename)の m です。今いるブランチの名前を変更します。

### Q. main に戻りたい

```bash
git switch main
```

戻ってきたら、もう一度 `git switch <さっきのブランチ名>` で作業ブランチに移れます。Tab キーで補完が効くので、最初の数文字 + Tab で入力が楽になります。

### Q. fatal: A branch named '...' already exists

すでに同じ名前のブランチがある状態で `-c` で作ろうとするとこのエラーが出ます。`git branch` で確認して、既にあれば `git switch <名前>` で移るだけで OK です(`-c` は不要)。

## まとめ

- `git switch -c <ブランチ名>` で新しいブランチを作って移動できる
- `git branch` で今いるブランチが分かる
- `main` で直接作業しないのは、他の人と独立に作業し、失敗してもやり直せるようにするため

次の章では、このブランチ上で実際に自己紹介ファイルを作って commit します。
