# 章立てメモ(運営用)

Zenn には公開しない、章立て設計のメモです。各章の **狙い**・**到達ゴール**・**分量目安**・**当日かける時間** をまとめています。

## 全体構成

| # | 章タイトル | 種別 | 当日時間 | 読了目安 | 到達ゴール |
|---|---|---|---|---|---|

## 章ごとの執筆方針

- **概念図はスライドに任せる**: Book ではテキスト主体でいい。図はスライドへの参照で済ませる。
- **コマンドは1ブロックずつ**: 連続コマンドの羅列はしない。1つずつ「やる」「結果を確認する」を交互にする。
- **Windows 注記**: コマンドが Windows で挙動が変わるところは Git Bash 前提と明記。`cp` / `mkdir -p` などは Git Bash で動く前提。
- **必須章は「やる手順」、参考章は「読み物」**: 文体を切り替える。
- **「なぜ」を必ず添える**: コマンドだけでなく、なぜそれをするのかを必ず書く(運営方針)。

## 想定する受講者像

- プログラミング学習中の専門学校生
- ターミナル操作はおぼつかない人もいる
- Git/GitHub は名前を知っているが実体は分からないレベル
- Windows 受講者が多数いる(Git Bash 前提)
